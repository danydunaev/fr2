
# Приложение заметок (Offline Notes App)

Простое веб-приложение для управления заметками с поддержкой офлайн-режима (Service Worker), архитектурой App Shell и возможностью установки как PWA (Progressive Web App).

## 📋 Описание

Приложение демонстрирует:
- ✅ Работу с `localStorage` для хранения данных
- ✅ Регистрацию и работу **Service Worker**
- ✅ Кэширование статических ресурсов для офлайн-доступа
- ✅ **App Shell архитектуру** — мгновенная загрузка каркаса
- ✅ **Динамическую загрузку контента** (страницы подгружаются по запросу)
- ✅ **HTTPS** для безопасной работы PWA
- ✅ Web App Manifest для установки приложения на устройство
- ✅ Полную функциональность в офлайн-режиме

## 🚀 Технологии

- **HTML5** — разметка
- **CSS3** — стилизация (тёмная тема, адаптив, анимации)
- **JavaScript (ES6+)** — логика приложения
- **Service Worker API** — офлайн-функциональность
- **App Shell Architecture** — мгновенная загрузка интерфейса
- **Web App Manifest** — установка как PWA
- **HTTPS** — безопасное соединение (mkcert)

## 🔧 Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/danydunaev/fr2
cd fr2/notes-app
```

### 2. Установка HTTPS сертификатов (для PWA)

```bash
# Установка mkcert (macOS)
brew install mkcert

# Создание сертификатов
mkcert -install
mkcert localhost 127.0.0.1 ::1
```

### 3. Запуск HTTPS-сервера на порту 3002

```bash
# Установка http-server
npm install -g http-server

# Запуск на порту 3002
http-server --ssl --cert localhost.pem --key localhost-key.pem -p 3002
```

### 4. Открытие приложения

```
https://localhost:3002
```

> **Важно:** Service Worker работает только по HTTPS. При запуске через HTTP (например, Live Server) SW не зарегистрируется.

## 🏗️ App Shell архитектура

### Что такое App Shell?
App Shell (каркас приложения) — это минимальный набор HTML, CSS и JavaScript, который обеспечивает базовую структуру интерфейса (шапка, навигация, контейнер). Этот каркас кэшируется при первом посещении и затем загружается мгновенно даже при плохом соединении. Внутрь контейнера динамически подгружается контент.

### Структура App Shell
- `index.html` — каркас (шапка, навигация, контейнер, статус подключения)
- `content/home.html` — динамический контент главной страницы (форма и список заметок)
- `content/about.html` — динамический контент страницы "О приложении"

### Стратегии кэширования в Service Worker

| Тип ресурса | Стратегия | Описание |
|-------------|-----------|----------|
| App Shell (index.html, app.js, иконки) | **Cache First** | Сначала из кэша, затем из сети |
| Динамический контент (content/*.html) | **Network First** | Сначала из сети, затем из кэша (fallback) |

## 🔐 HTTPS для PWA

Service Worker требует HTTPS для работы. На локальной машине мы используем **mkcert** для создания доверенного самоподписанного сертификата.

### Проверка HTTPS
1. Откройте `https://localhost:3002`
2. В адресной строке должен отображаться **замочек** 🔒
3. DevTools → **Security** → статус "Secure"

## 🔐 Service Worker

### Два кэша
| Кэш | Назначение |
|-----|------------|
| `app-shell-v1` | Статические ресурсы (App Shell) |
| `dynamic-content-v1` | Динамические страницы (content/*) |

### Что кэшируется в App Shell
```javascript
const SHELL_ASSETS = [
    '/',
    '/index.html',
    '/app.js',
    '/manifest.json',
    '/icons/*.png'
];
```

### Стратегии обработки запросов
| Тип запроса | Стратегия |
|-------------|-----------|
| App Shell (SHELL_ASSETS) | Cache First |
| Динамический контент (/content/*) | Network First |
| Остальные запросы | Network First с fallback на кэш |

### События Service Worker
| Событие | Описание |
|---------|----------|
| `install` | Кэширование App Shell, вызов `skipWaiting()` |
| `activate` | Очистка старых кэшей, вызов `clients.claim()` |
| `fetch` | Перехват запросов с разными стратегиями |

## 🧪 Тестирование

### Проверка Service Worker (Chrome/Edge)
1. Откройте DevTools (`F12` или `⌘+⌥+I`)
2. Перейдите на вкладку **Application** → **Service Workers**
3. Убедитесь, что SW активен (status: activated and running)
4. Перейдите в **Cache Storage** → должны быть два кэша:
   - `app-shell-v1` — статические файлы
   - `dynamic-content-v1` — динамические страницы

### Проверка HTTPS
1. DevTools → **Security** → статус "Secure"
2. В адресной строке — зелёный замочек 🔒

### Проверка Manifest (Chrome/Edge)
1. DevTools → **Application** → **Manifest**
2. Проверьте, что все поля отображаются, иконки загружены

### Тестирование офлайн-режима
1. Откройте приложение, добавьте несколько заметок
2. DevTools → **Network** → выберите **Offline**
3. Обновите страницу (`⌘+R`)
4. **App Shell** загрузится мгновенно из кэша
5. Нажмите на вкладки — контент подгрузится из кэша
6. Добавьте новую заметку — сохранится в localStorage
7. Включите сеть — данные сохранятся

### Тестирование производительности
1. DevTools → **Network** → **Throttling** → **Slow 3G**
2. Обновите страницу
3. Каркас (шапка, меню) появится сразу
4. Контент подгрузится чуть позже

## 🔍 Отладка

### Частые проблемы и решения

| Проблема | Решение |
|----------|---------|
| SW не регистрируется | Проверьте, что приложение открыто по HTTPS |
| Ошибка "Not Secure" | Установите сертификаты через mkcert |
| Иконки не отображаются | Проверьте пути в manifest.json и index.html |
| Нет кнопки установки | Используйте Chrome/Edge, проверьте SW активен |
| Динамический контент не загружается офлайн | Проверьте Network First стратегию в sw.js |
| Ошибка 404 на иконки | Убедитесь, что иконки есть в папке icons/ |
| Двойной кэш | В activate событии удаляются старые версии |

### Полезные команды DevTools
- `⌘+⌥+I` — открыть DevTools
- `⌘+R` — перезагрузка страницы
- `⌘+Shift+R` — жесткая перезагрузка (очистка кэша)
- Application → Service Workers → **Update** — принудительное обновление SW
- Application → Service Workers → **Unregister** — удаление SW
